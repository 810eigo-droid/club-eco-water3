## 🏗 ファイル構成と「2系統」の関係（最重要・必読）

このプロジェクトには **(A) モジュール版（編集用ソース）** と **(B) スタンドアロン版（公開用）** の2つがあります。

```
club-eco-water/                 ← (A) 編集用ソース（source of truth）
├─ index.html                   HTML本体（css/jsを外部参照）
├─ css/style.css                デザイン一式
├─ js/app.js                    スクロール表示・固定CTAバー
├─ images/                      画像置き場（現状は空＝仮置き枠表示）
├─ README.md                    制作メモ・画像仕様
└─ HANDOFF.md                   ← このファイル

club-eco-water-standalone/      ← (B) 公開用（自動生成物）
└─ index.html                   (A)のcss/jsを1ファイルに埋め込んだ版
```

### なぜ2系統なのか
- クライアントは **GitHubのWebアップロード**で公開している（VSCode/Git操作に不慣れ）。
- フォルダ構成（css/, js/）が崩れて **CSSが効かない事故が起きた**ため、
  **公開は「CSS・JSを全部埋め込んだ単一ファイル(B)」を index.html として1枚アップロードする運用**に切り替えた。
- → **編集は必ず (A) モジュール版で行い、最後に (B) を再生成**する。直接 (B) を手編集しない。

### (B) スタンドアロン版の再生成コマンド
`club-eco-water/` で以下を実行すると `club-eco-water-standalone/index.html` が更新される：

```python
# python3 で実行（club-eco-water/ をカレントに）
html = open('index.html', encoding='utf-8').read()
css  = open('css/style.css', encoding='utf-8').read()
js   = open('js/app.js', encoding='utf-8').read()
html = html.replace('<link rel="stylesheet" href="css/style.css">', "<style>\n"+css+"\n</style>")
html = html.replace('<script src="js/app.js"></script>', "<script>\n"+js+"\n</script>")
open('../club-eco-water-standalone/index.html','w',encoding='utf-8').write(html)
```

> 注意：(A) の `<head>` 内の `<link rel="stylesheet" href="css/style.css">` と
> 末尾の `<script src="js/app.js"></script>` の表記を変えると、上の置換が効かなくなる。変更時は同期すること。
